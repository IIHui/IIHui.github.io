views

创建: 2021-05-28   |   更新: 2021-05-28   |   分类: iOS     |   字数: 3405   |   阅读: 7 分钟

视图层次结构是视图组织的主要模式。一个视图可以有多个子视图，但只有一个直接父视图。若一个视图被从界面中被移除，则它的子视图也会被移除；如果一个视图被隐藏，则它的子视图都会被隐藏；如果一个视图被移动，它的子视图也会随之移动。也就是说，一个视图的变化会与作用到它的子视图中。

UIView是UIResponder的子类，即它不仅是用户能看到的界面，也是用户交互所用的界面。例如点击或者滑动，也就是说视图层次结构也是响应者链的基础，尽管它与响应者链不完全相同。

一个视图的创建可能来自nib，也可能来自代码。不能说绝对的说哪种方式更好一些，这取决于个人需求和倾向以及应用程序的整体架构。

window 和 root View

view的最顶层视图是window。window是UIWindow或者其子类的实例，UIWindow也是UIView的子类。一个可见的window构成了所有其他可见view的背景，也是所有可见视图的superview。一个view如果是可见的，那么它一定是某个可见window的子视图。

在编程中，我们不会手动或者直接用子视图来填充window，那是因为window和它所包含的视图之间的连接是是根视图控制器rootViewController。一旦一个viewController被实例化，并且赋值给window的rootViewController，则这个viewController的main view则会占据整个window。

这个main view是window的唯一子视图，其他所有可见的视图都是根视图控制器main view的子视图（在一定深度上）。

window scene architecture

从iOS 13开始App的window是由UIWindwScene来提供，这与iOS 12以前的主架构最大的不同。

在iOS 12以前，是基于window的架构，window是AppDelegate的一个属性，如果项目是在XCode 10或者之前创建的，则使用的是这种老架构。

在iOS 13以后，是基于scene的架构，window是scene delegate的一个属性，如果项目是在XCode 11或之后创建的，则使用的是这种新架构。

App Launches

应用程序最终包括对UIApplicationMain函数的一次调用。与Objective-C不同，一个典型的Swift项目不会在代码中明确地进行这一调用，它是在幕后为你调用的。

iOS 13

使用storyboard

  1. UIAppplicationMain首先实例化UIApplication，并保留这个实例，作为共享的实例，在代码中可以使用UIApplication.shared来访问。接着根据@UIApplicationMain或@main来实例化AppDelegate，并保留这个实例，使得它在应用程序中的整个生命周期中都存在，在代码中可以使用UIApplication.shared.delegate来访问。

  2. 接着UIApplicationMain会调用AppDelegate的application(_:didFinishLaunchingWithOptions:)方法。

  3. UIAppplicationMain创建UISceneSession和UIWindowScene以及UIWindwScene的delegate。

  4. UIAppplicationMain查看初始scene是否使用了storyboard，即在info.plist中是否有Application Scene Manifest，若有则对storybaord中的初始化viewController进行初始化。

  5. 如果scene使用了storyboard，UIAppplicationMain则会实例化一个UIWindow并将这个实例赋值给scene的delegate的window属性。

  6. 如果scene使用了storyboard，则将storyboard中的初始化的viewController赋值给window实例的属性rootViewController。此时，初始ViewController的view变成了Window的唯一子视图。

  7. UIApplicationMain通过调用UIWindow实例方法makeKeyAndVisible，使你的应用程序的界面出现。

  8. scene的(_:willConnectTo:options:)被调用，给你更多的代码以运行的机会。

Subview and Superview

Color

视图可以通过其backgroundColor属性指定一个背景颜色，如果一个视图的背景色是nil，则该视图有一个透明的背景色。

可见性和透明度

view的可见性和不透明度跟3个属性相关，他们分别是isHidden，alpha，isOpaque。

isHidden

视图。视图可以通过将其isHidden属性设置为true而变得不可见，并通过将其设置为false 而再次可见。被隐藏的视图（通常）不会收到触摸事件，所以对用户来说，就像该视图不存在一样。

alpha

一个视图可以通过其alpha属性使其部分或完全透明。alpha属性值可以在0～1.0，1.0表示完全不透明，0.0表示透明。alpha属性既影响view的背景色也会影响其内容的透明度。

如果一个视图显示了一个图像，并且有一个背景颜色，而它的alpha值小于1.0，那么背景颜色就会渗入到图像中。如果一个父试图的alpha值为0.5，则它的所有子视图的alpha都小于0.5，因为无论它们的alpha值是多少，都会相对于0.5绘制的。

一个完全透明（或非常接近透明）的视图就像一个isHidden为真的视图：它和它的子视图都是不可见的，而且（通常）不能被触摸到。

为了使事情更加复杂，颜色也有一个alpha值。一个视图的alpha值可以是1.0，但仍然有一个透明的背景，因为它的背景颜色的alpha值小于1.0。

isOpaque

isOpaque属性不同于isHidden和alpha，改变isOpaque属性值对视图的外观没有影响，只是对视图系统的提示。

如果一个试图完全由不透明的物质填充并且它的alpha属性设置为1.0，那么视图就是完全不透明的，此时如果将视图的isOpaque属性设置为true来告知绘图系统这一事实，就可以更有效地进行绘制（对性能的拖累更小）。

当设置isHidden和alpha的值时，isOpaque属性值并不会跟着变化。isOpaque属性值的设置完全取决你自己，它的默认值为true。

Frame

每个view由一个frame属性（是一个矩形CGRect），表示矩形在父视图的坐标系统中的位置。默认情况下，父视图的坐标系将以左上角为原点，X坐标向右正向增长，Y坐标向下正向增长。

设置视图的frame值既可以重新改变视图的位置，也可以调整的大小，或者同时改变位置和大小。视图的frame属性如果是CGRect.zero的话，则视图实际上是不可见的。

如果一个视图有一个你希望它采用的标准尺寸，特别是相对于它的内容（比如UIButton相对于它的标题），另一个办法是调用其sizeToFit方法。

如果一个视图使用nib进行初始化的时候，那么不会调用init(frame:)，调用的是init(coder:)。

Bounds and Center

Bounds

假设有一个父视图和子视图，子视图需要10个点的嵌入，如下图所示，首先想到的是设置子视图的frame，但是怎么设置？

CGRect有个insetBy(dx:dy:)方法可以把一个矩形很容易的嵌入到另一个矩形中。描述视图在其自身坐标中的矩形的CGRect是视图的bounds属性。

如果改变一个视图的bounds属性的size值，则会影响到其frame属性的size值。视图frame属性size的变化是围绕其中心点进行的，即中心点保持不变：

let v1 = UIView(frame:CGRect(113, 111, 132, 194)) 
v1.backgroundColor = UIColor(red: 1, green: 0.4, blue: 1, alpha: 1) 
let v2 = UIView(frame:v1.bounds.insetBy(dx: 10, dy: 10)) 
v2.backgroundColor = UIColor(red: 0.5, green: 1, blue: 0, alpha: 1)
self.view.addSubview(v1) 
v1.addSubview(v2) 
v2.bounds.size.height += 20 
v2.bounds.size.width += 20

如果创建了一个视图，其bounds坐标系的零点(0.0,0.0)在左上角。因为一个子视图是相对于它的父视图的坐标系定位的，所以父视图的bounds原点的改变会导致子视图位置的变化：

let v1 = UIView(frame:CGRect(113, 111, 132, 194)) 
v1.backgroundColor = UIColor(red: 1, green: 0.4, blue: 1, alpha: 1) 
let v2 = UIView(frame:v1.bounds.insetBy(dx: 10, dy: 10)) 
v2.backgroundColor = UIColor(red: 0.5, green: 1, blue: 0, alpha: 1) 
self.view.addSubview(v1) 
v1.addSubview(v2) 
v1.bounds.origin.x += 10 
v1.bounds.origin.y += 10

center

同样地，若改变一个视图的frame属性的size值，也会改变其bounds属性的size值。但是，改变bounds的size大小时，不会改变其center属性。
